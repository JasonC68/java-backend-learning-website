# 跨设备云同步 · 部署说明

要在**所有设备访问 + 进度同步**，需要两件免费的事：
**① 把网页放到网上拿一个网址；② 用一个云仓库存进度。**
全程免费，约 10 分钟，不用写代码。

---

## 第一步：建云仓库存进度（JSONBin，约 3 分钟）

1. 打开 https://jsonbin.io ，用邮箱/Google 免费注册登录。
2. 进入控制台点 **Create Bin**（创建 Bin）。
3. 在内容框里删掉默认内容，只填一个空对象：

   ```
   {}
   ```

4. 点 **Create / Save**，创建成功后：
   - 在地址栏或 Bin 详情里复制 **Bin ID**（一串字母数字，形如 `6650a1b2ac1f...`）。
   - 左侧菜单 **API Keys** 里复制 **Master Key**（`$2a$10$...` 开头的一长串）。
5. 把这两个值记下来，第三步要用。

> 这个 Bin 就是你进度的"云端档案"，所有设备读写同一个 Bin 实现同步。

---

## 第二步：把网页放到网上（Netlify Drop，约 3 分钟）

1. 打开 https://app.netlify.com/drop 。
2. 把文件 **`秋招必背打卡表-云同步.html`** 直接拖进网页里的虚线框。
3. 上传完会得到一个网址（形如 `https://xxx-yyy.netlify.app/秋招必背打卡表-云同步.html`）。
   - 建议注册免费账号（可用 Google），这样网址会长期保留、还能改名。
   - 想换更短网址：Site settings → Change site name。
4. 把这个网址**收藏到所有设备**（手机加到主屏更方便）。

> 也可以用 GitHub Pages、Cloudflare Pages、Vercel 等，原理一样：让 html 有个公网网址。

---

## 第三步：每台设备配置一次同步（约 1 分钟/台）

1. 用上面的网址打开页面。
2. 点右上区域的 **☁️ 云同步设置**。
3. 把第一步的 **Bin ID** 和 **Master Key** 粘进去，点 **保存并同步**。
4. 右上角小标签变成 **"已同步 + 时间"** 就成功了。

之后：

- 在任意设备点掌握程度 / ＋ / − ，进度会自动上传（标签显示"待同步→已同步"）。
- 换另一台设备打开，会自动**拉取最新**；也可手动点 **⬇️ 拉取**。
- 偶尔多设备同时改，以最后一次上传为准（单人使用基本不冲突）。换设备前先点一下「拉取」最稳妥。

---

## 常见问题

- **标签显示"同步失败"**：多半是 Bin ID 或 Master Key 填错，重开「云同步设置」核对；或检查网络。
- **数据安全**：Master Key 写在你自己的浏览器里。这是个人学习进度，风险很低；但别把带 Key 的网址截图公开分享。
- **想换存储**：JSONBin 免费额度对单人足够；若想更稳，可改用 Firebase/Supabase（需要我再改一版页面）。
- **离线也能用**：没网时照常打卡，存在本地；联网后点「⬆️ 上传」补传即可。

---

## 一句话流程

注册 JSONBin 建 Bin → 拿 Bin ID + Master Key → 把 html 拖到 Netlify 拿网址 → 每台设备打开网址、填一次配置 → 全设备同步完成。
